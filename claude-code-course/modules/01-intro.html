Claude Code 是什么？

想象你在终端里打字："帮我添加一个登录功能"，然后 AI 就开始写代码、运行测试、修复 bug——直到功能完成。

这就是 Claude Code：一个让 AI 成为你编程助手的命令行工具。

claude-code
$ claude
你: 帮我添加一个用户登录功能
Claude: 好的，让我先看看现有的用户系统...
→ 正在读取 src/auth/user.ts
→ 正在创建 src/auth/login.ts
→ 正在运行测试...
✓ 所有测试通过！登录功能已完成。

但它不只是写代码。它可以读取文件、执行命令、搜索代码、运行测试、创建 PR——所有你在开发中需要做的事情。

💡 核心洞察

Claude Code 不是一个简单的聊天机器人。它是一个AI Agent——能够自己决定需要做什么、调用哪些工具、如何完成任务。

它是用什么构建的？

Claude Code 使用现代 JavaScript 生态系统，但选择了一些独特的技术：

⚡

Bun

超快的 JavaScript 运行时，比 Node.js 快 3-4 倍

替代 Node.js
⚛️

React + Ink

用 React 组件构建终端 UI，不是网页

终端中的 React
📘

TypeScript

类型安全的 JavaScript，1900+ 文件

512K+ 行代码
🔍

ripgrep

极速代码搜索工具

Rust 驱动

这个技术栈的选择都是为了性能——启动要快、搜索要快、响应要快。开发者不想等待。

这段代码在做什么？

1-3

注释解释了为什么这些代码必须最先运行——它们是性能优化。

6-7

记录一个时间戳，用来测量启动性能。

8-10

立即启动读取 MDM 设置（企业管理设置），这样它可以在后台运行。

11-16

立即启动从钥匙串读取密码，也在后台运行。

17-19

导入其他需要的模块：特性开关、命令行解析器、React。

🎯 为什么要这样做？

注意到代码在导入其他东西之前，就启动了两个耗时操作？这叫并行预取——在主程序加载的同时，让慢操作在后台运行。结果：启动时间减少 60+ 毫秒。
