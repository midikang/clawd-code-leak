核心对话：Anthropic API

当你向 Claude Code 提问时，它需要调用 Anthropic 的 API 来获取 AI 的响应。这个过程发生在 QueryEngine.ts 中。

src/services/api/claude.ts 
// 导入 Anthropic SDK
import type {
  BetaMessage,
  BetaMessageStreamParams,
  BetaRawMessageStreamEvent,
  MessageParam,
} from '@anthropic-ai/sdk/resources/beta/messages/messages.mjs'

// 调用 API 的核心函数
export async function callClaudeAPI(
  messages: MessageParam[],
  systemPrompt: string,
  tools: Tool[],
  model: string
): Promise> {
  // 构建 API 请求参数
  const params: BetaMessageStreamParams = {
    model,
    max_tokens: getModelMaxOutputTokens(),
    messages,
    system: systemPrompt,
    tools: tools.map(toolToAPISchema),
    stream: true,
  }

  // 发送流式请求
  return anthropic.messages.stream(params)
}

定义函数签名：接收消息历史、系统提示、工具列表和模型名称。

构建 API 请求：包含对话历史、工具定义、最大输出长度等参数。

26

使用 流式 API——AI 的响应会一点点返回，而不是等待全部完成。

🎯 为什么用流式 API？

想象你在等待一个很长的回答。流式 API 让你可以边读边等——每收到一个词就显示出来，而不是等待整个回答完成。这让体验更流畅。

连接外部工具：MCP

Model Context Protocol (MCP) 是一个让 AI 应用连接外部工具的标准协议。想象它是一个"插座"——你可以插入各种工具服务器。

src/services/mcp/client.ts 
import { Client } from '@modelcontextprotocol/sdk/client/index.js'
import { StdioClientTransport } from '@modelcontextprotocol/sdk/client/stdio.js'

// 连接到 MCP 服务器
export async function connectToMCPServer(
  serverConfig: McpServerConfig
): Promise {
  // 创建传输层（stdio = 标准输入输出）
  const transport = new StdioClientTransport({
    command: serverConfig.command,
    args: serverConfig.args,
    env: serverConfig.env,
  })

  // 创建客户端
  const client = new Client({
    name: 'claude-code',
    version: '1.0.0',
  })

  // 连接
  await client.connect(transport)

  // 列出服务器提供的工具
  const tools = await client.listTools()

  return { client, tools }
}

💡 为什么需要 MCP？

Claude Code 自带的工具是固定的（读文件、执行命令等）。但你可能想让它访问你的系统——比如你的数据库、Jira、内部 API。MCP 让你可以轻松添加这些能力，而不需要修改 Claude Code 的源代码。

理解代码：LSP

Language Server Protocol (LSP) 是微软创造的协议，让编辑器能够获得代码智能功能——比如"跳转到定义"、"查找引用"、类型提示等。

Claude Code 使用 LSP 来更好地理解代码结构。

安全认证：OAuth

Claude Code 需要证明"你是谁"才能调用 Anthropic API。这通过 OAuth 2.0 协议完成——一个业界标准的授权流程。

src/constants/oauth.ts 
// OAuth 授权范围
export const CLAUDE_AI_OAUTH_SCOPES = [
  'user:profile',           // 读取用户资料
  'user:inference',         // 调用 AI 推理 API
  'user:sessions:claude_code', // 会话管理
  'user:mcp_servers',       // MCP 服务器配置
  'user:file_upload',       // 文件上传
] as const

// OAuth 配置
export function getOauthConfig() {
  return {
    clientId: 'claude-code-cli',
    authUrl: 'https://auth.anthropic.com/oauth/authorize',
    tokenUrl: 'https://auth.anthropic.com/oauth/token',
    scopes: CLAUDE_AI_OAUTH_SCOPES,
    redirectUri: 'http://127.0.0.1:8910/callback',
  }
}

客户端 ID：标识这是"Claude Code CLI"应用。

14-15

授权和令牌 URL：Anthropic 的 OAuth 服务器地址。

17

回调 URL：用户授权后，浏览器会跳转到这个本地地址，Claude Code 在这里监听结果。

💡 为什么用 OAuth 而不是 API Key？

API Key 是固定的密码——如果泄露就很危险。OAuth token 可以定期刷新、可以限制权限范围、可以随时撤销。这更安全，也是现代应用的标准做法。
